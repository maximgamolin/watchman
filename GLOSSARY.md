# Глоссарий предметной области — Watchman

Каждому понятию предметной области соответствует **одно** английское слово.
Синонимы запрещены. При добавлении нового понятия — сначала проверь глоссарий.

---

## Сущности предметной области

| Термин | Тип | Описание |
|---|---|---|
| `GroupMember` | Entity | Участник Telegram-группы. Единица мультитенантности: один пользователь может быть участником многих групп — каждая пара `(user_id, group_id)` является отдельным `GroupMember`. |
| `CaptchaSession` | Value Object (Redis) | Активная сессия капчи: число, которое должен ввести пользователь, ID сообщений и время создания. Хранится в Redis, не в БД. |
| `DeletedMessage` | Record (audit) | Запись об удалённом сообщении пользователя. Хранится в БД для аудита. Не является доменным агрегатом — только лог. |

---

## Типы-значения (`NewType`)

| Термин | Базовый тип | Описание |
|---|---|---|
| `UserID` | `int` | Telegram `user.id`. Уникален глобально в рамках Telegram. |
| `GroupID` | `int` | Telegram `chat.id` группы или супергруппы. Отрицательное число. |
| `MessageID` | `int` | Telegram `message_id` внутри конкретного чата. Уникален только в пределах одного чата. |

---

## Словарь бизнес-терминов

| Термин | Английское слово | Описание |
|---|---|---|
| Капча | `captcha` | Проверка нового участника группы: бот просит ввести случайное число от 0 до 9. |
| Сессия капчи | `captcha_session` | Временный объект в Redis, связывающий пользователя с ожидаемым числом и ID сообщений. TTL = 60 секунд. |
| Участник группы | `group_member` | Пользователь, который писал хотя бы одно сообщение в группе. Создаётся в момент первого сообщения. |
| Прошёл капчу | `captcha_passed` | Состояние `GroupMember`, при котором пользователь ввёл верное число. Только такие пользователи допускаются к переписке. |
| Удалённое сообщение | `deleted_message` | Сообщение, удалённое ботом: либо из-за истечения капчи, либо из-за спама в ожидании капчи. |
| Ожидаемое число | `expected_number` | Случайное целое от 0 до 9, которое пользователь должен ввести для прохождения капчи. |
| Истечение капчи | `captcha_expired` | Причина удаления сообщения: пользователь не ввёл число в течение 60 секунд. |
| Спам во время капчи | `spam_during_captcha` | Причина удаления сообщения: пользователь продолжал писать, пока капча была активна. |
| Оригинальное сообщение | `original_message` | Первое сообщение пользователя, на которое бот ответил капчей (reply). |
| Сообщение с капчей | `captcha_message` | Сообщение бота в группе с просьбой ввести число. Удаляется после истечения TTL. |

---

## Причины удаления сообщений (`reason`)

Используются в таблице `deleted_message`. Закрытый список — новые значения согласуются явно.

| Значение | Когда присваивается |
|---|---|
| `captcha_expired` | Celery-задача обнаружила просроченную сессию капчи (прошло > 60 сек). Удалено сообщение бота, в лог записано исходное сообщение пользователя. |
| `spam_during_captcha` | Пользователь написал новое сообщение пока капча ещё активна (неверный ответ или произвольный текст). Сообщение удалено немедленно. |

## Ключи Redis

| Паттерн | Формат | Описание |
|---|---|---|
| `captcha:{group_id}:{user_id}` | `captcha:-100123456789:67890` | Сессия капчи. Значение — JSON с полями `expected_number`, `captcha_message_id`, `original_message_id`, `original_message_text`, `user_id`, `group_id`, `created_at`. |

---

## Таблицы БД

| Таблица | Сущность | Описание |
|---|---|---|
| `group_member` | `GroupMember` | Участники групп. Пара `(user_id, group_id)` — натуральный ключ. Запись создаётся при первом сообщении пользователя. |
| `deleted_message` | — | Аудит-лог удалённых сообщений. Только добавление, без обновления и удаления. |

---

## Правила именования

- **Один термин — одно слово.** Нельзя писать `member` в одном месте и `user` в другом для одного понятия.
- **`GroupMember`, не `User`.** В контексте бота пользователь всегда рассматривается в паре с группой.
- **`captcha_session`, не `captcha_task` / `captcha_request`.** Сессия — временный объект с состоянием.
- **`expected_number`, не `correct_number` / `answer` / `digit`.** Число задано заранее ботом, пользователь его вводит.
- **`deleted_message`, не `removed_message` / `blocked_message`.** Физически удалено через Telegram API.
